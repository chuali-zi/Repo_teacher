## <a id="ix-01"></a>IX-01 全局交互模型

### 交互范式

Repo Tutor 是一个**对话驱动的教学产品**，不是仪表盘型产品。核心交互模式：

```
用户输入仓库 → 系统分析 → 首轮教学报告 → 用户追问 → Agent 回答并引导 → 循环
```

### 核心交互循环

```
┌─────────────────────────────────────────────┐
│                                             │
│   [空闲态] ──输入仓库──→ [分析中]            │
│       ↑                      │              │
│       │                      ↓              │
│   [切换仓库]           [首轮报告展示]         │
│       ↑                      │              │
│       │                      ↓              │
│       │              ┌─→ [对话态] ←─┐       │
│       │              │      │       │       │
│       └──────────────┘      ↓       │       │
│                        [用户追问]    │       │
│                             │       │       │
│                             ↓       │       │
│                        [Agent回答]──┘       │
│                                             │
└─────────────────────────────────────────────┘
```

### 交互原则

| 编号 | 原则 | 说明 |
|------|------|------|
| P1 | Agent 主动带路 | 每轮结束必须有"下一步建议"，用户永远知道可以做什么 |
| P2 | 连续讲解优先 | 不频繁打断用户，不设阻塞式小测验 |
| P3 | 单轮可消化 | 每轮 2-4 个核心认知点，不信息轰炸 |
| P4 | 自然语言为主 | 图示/ASCII 图仅作辅助 |
| P5 | 骨架先于细节 | 默认从宏观到微观，用户可打破顺序 |

## <a id="ix-02"></a>IX-02 关键页面定义

本产品共有 **3 个关键视图**，均为对话式布局的不同阶段状态。

### 视图 A：仓库输入视图（空闲态）

```
┌──────────────────────────────────────────────┐
│  Repo Tutor                                  │
│                                              │
│  ┌────────────────────────────────────────┐  │
│  │ 粘贴仓库路径或 GitHub URL              │  │
│  │                                        │  │
│  │  [输入框]                         [开始]│  │
│  └────────────────────────────────────────┘  │
│                                              │
│  示例：                                       │
│  · /home/user/my-project                     │
│  · https://github.com/owner/repo             │
│                                              │
│  支持 Python 仓库完整分析                      │
│  其他语言仓库提供基础结构总览                    │
│                                              │
└──────────────────────────────────────────────┘
```

**交互要素**：
- 输入框：单行文本输入，支持粘贴
- 输入即时校验：格式合法性（路径格式 / URL 格式）
- 开始按钮：输入非空且格式合法时可点击
- 输入框下方：简要说明支持的输入类型

### 视图 B：分析进行中视图

```
┌──────────────────────────────────────────────┐
│  Repo Tutor                                  │
│                                              │
│  正在分析仓库：owner/repo                     │
│                                              │
│  [████████░░░░░░░░░░░░]                      │
│                                              │
│  · ✓ 仓库接入成功                             │
│  · ✓ 文件树扫描完成                           │
│  · ● 正在识别入口与模块...                     │
│  · ○ 分析依赖来源                             │
│  · ○ 组织教学骨架                             │
│                                              │
│  小型仓库通常 30 秒内完成                      │
│                                              │
└──────────────────────────────────────────────┘
```

**交互要素**：
- 进度指示器（非精确百分比，而是阶段性步骤）
- 步骤清单：已完成 ✓ / 进行中 ● / 等待中 ○
- 大型仓库时显示"已启用降级分析"提示
- 无取消按钮（分析为快速操作，不需中断）

### 视图 C：教学对话视图（核心视图）

```
┌──────────────────────────────────────────────┐
│  Repo Tutor · owner/repo          [切换仓库] │
│─────────────────────────────────────────────-│
│                                              │
│  ┌── Agent 首轮报告 ──────────────────────┐  │
│  │                                        │  │
│  │  [首轮教学报告内容]                      │  │
│  │  (详见 IX-08 输出展示规格)               │  │
│  │                                        │  │
│  │  📎 下一步建议：                         │
│  │  · 想了解启动流程怎么走？                 │
│  │  · 想看看核心模块的职责？                 │
│  │  · 想深入某个目录？                      │  │
│  └────────────────────────────────────────┘  │
│                                              │
│  ┌── 用户 ────────────────────────────────┐  │
│  │  启动流程怎么走？                        │  │
│  └────────────────────────────────────────┘  │
│                                              │
│  ┌── Agent ───────────────────────────────┐  │
│  │  [本轮重点]                             │  │
│  │  [直接解释]                             │  │
│  │  [与整体的关系]                          │  │
│  │  [证据]                                │  │
│  │  [不确定项]                             │  │
│  │  [下一步建议]                           │  │
│  └────────────────────────────────────────┘  │
│                                              │
│  ┌────────────────────────────────────────┐  │
│  │ 输入你的问题...                    [发送]│  │
│  └────────────────────────────────────────┘  │
│                                              │
└──────────────────────────────────────────────┘
```

**交互要素**：
- 顶栏：当前仓库名称 + 切换仓库按钮
- 对话区：消息流式布局，自动滚动到最新
- Agent 消息气泡内：结构化内容分区（详见 IX-08）
- 下一步建议：可点击的建议项（点击即填入输入框并发送）
- 输入框：多行文本输入，Enter 发送，Shift+Enter 换行
- Agent 正在回答时：输入框置灰，显示"Agent 正在思考..."

## <a id="ix-03"></a>IX-03 主流程：首次仓库接入

### 流程图

```
用户打开产品
    │
    ▼
[视图A：空闲态]
    │
    │ 用户输入仓库路径或 URL
    ▼
┌─ 即时格式校验 ─┐
│                │
│ 不合法         │ 合法
▼                ▼
输入框下方       用户点击"开始"
显示格式提示         │
(红色文字)          ▼
                [视图B：分析中]
                    │
            ┌───────┼───────┐
            │       │       │
         接入失败  接入成功  超时
            │       │       │
            ▼       ▼       ▼
         错误提示  分析进行  超时提示
         (见IX-09) 步骤展示  (见IX-09)
                    │
                    ▼
                分析完成
                    │
            ┌───────┴───────┐
            │               │
         Python仓库     非Python仓库
            │               │
            ▼               ▼
         完整首轮报告    降级首轮报告
         (OUT-1完整)   (OUT-12降级)
            │               │
            └───────┬───────┘
                    ▼
            [视图C：教学对话态]
```

### 步骤细节

| 步骤 | 用户行为 | 系统响应 | 界面变化 |
|------|---------|---------|---------|
| 1 | 输入仓库路径/URL | 即时格式校验 | 合法则开始按钮可用；不合法则显示红色提示 |
| 2 | 点击"开始" | 切换到分析中视图 | 显示进度步骤清单 |
| 3 | 等待 | 逐步完成分析 | 步骤状态逐个从 ○ 变为 ● 再变为 ✓ |
| 4 | 阅读报告 | 输出首轮教学报告 | 切换到对话视图，首条消息为教学报告 |
| 5 | 点击建议或自由输入 | 进入多轮对话态 | 追加新的消息气泡 |

### 即时格式校验规则

| 输入内容 | 校验结果 | 提示文案 |
|----------|---------|---------|
| 空 | 开始按钮禁用 | 无提示 |
| 非路径非 URL 的任意文字 | 不合法 | "请输入本地仓库路径或 GitHub URL" |
| 以 `/` 或盘符开头的路径格式 | 合法（格式层面） | 无提示 |
| `https://github.com/owner/repo` 格式 | 合法 | 无提示 |
| GitHub URL 但非 `github.com` | 不合法 | "当前仅支持 github.com 公共仓库" |

## <a id="ix-04"></a>IX-04 主流程：多轮教学对话

### 对话循环

```
[首轮报告展示完毕]
        │
        ▼
┌─→ [等待用户输入] ←──────────────────────────┐
│       │                                      │
│       │ 用户发送消息                           │
│       ▼                                      │
│   [Agent 处理中]                              │
│       │                                      │
│       │ 显示"Agent 正在思考..."                │
│       ▼                                      │
│   [Agent 回答]                               │
│       │                                      │
│       ├── 回答内容（结构化）                   │
│       ├── 不确定项标注                        │
│       └── 下一步建议（可点击）─────────────────┘
│       │
│       │ 用户点击建议 / 自由输入
│       │
└───────┘
```

### Agent 回答结构（每轮固定结构）

每轮 Agent 回答按以下顺序组织，缺省项可省略但不可乱序：

```
1. 本轮重点          ← 一句话说明这轮在讲什么
2. 直接解释          ← 核心讲解内容（2-4 个认知点）
3. 与整体的关系      ← 当前内容在仓库全局中的位置
4. 证据或判断依据    ← 基于哪些文件/配置/import 得出结论
5. 不确定项          ← 当前无法确认的内容（可省略）
6. 下一步建议        ← 1-3 条可点击建议
```

### 对话中的上下文保持行为

| 场景 | 期望交互 |
|------|---------|
| 用户第 3 轮问"这个模块做什么" | Agent 基于前 2 轮已建立的上下文回答，不重新介绍仓库 |
| 用户第 4 轮问"上面说的入口，展开讲讲" | Agent 引用第 1 轮提到的入口候选，继续深入 |
| 用户第 5 轮问"总结一下目前学到什么" | Agent 回顾前 4 轮内容，生成阶段性总结 |
| 用户问的内容与之前无关 | Agent 正常回答，但保持仓库和学习目标不变 |

### 下一步建议的交互

- 每轮回答末尾提供 1-3 条下一步建议
- 建议以按钮/链接形式展示，用户可直接点击
- 点击后等同于用户输入该文字并发送
- 建议内容随对话推进而变化，不重复已讲解内容

## <a id="ix-05"></a>IX-05 主流程：学习目标切换

### 切换方式

用户通过自然语言在对话中显式切换，无需菜单或按钮：

```
用户："只看入口"
         │
         ▼
Agent 识别为目标切换指令
         │
         ▼
Agent 聚焦到目标主题
         │
         ├── 回答聚焦于该主题
         ├── 保持当前仓库不变
         ├── 保持之前的分析结果可引用
         └── 下一步建议围绕新目标
```

### 目标切换映射表

| 用户指令 | Agent 行为变化 | 回答聚焦内容 |
|----------|--------------|-------------|
| "只看结构" | 聚焦仓库结构总览 | 关键目录、模块组成、主路径 vs 支撑 |
| "只看入口" | 聚焦入口候选 | 入口文件、理由、置信度 |
| "只看启动流程" | 聚焦候选数据流/主流程 | 入口→模块→数据流向 |
| "只看某个模块" | 聚焦指定模块 | 模块职责、上下游、层级位置 |
| "只看依赖来源" | 聚焦 import 来源 | 内部/标准库/第三方/未知分类 |
| "只看分层" | 聚焦教学式分层 | 各层对应模块、主路径标注 |
| "总结一下" | 生成阶段性总结 | 回顾已讲解内容，梳理认知图谱 |
| "讲浅一点" | 降低讲解深度 | 见 IX-10 |
| "讲深一点" | 提升讲解深度 | 见 IX-10 |

### 切换后回答模板

```
[目标切换确认]  ← "好的，我们聚焦到入口候选。"
[聚焦内容讲解]  ← 按切换后的主题组织内容
[与之前的关联]  ← 简要说明与前几轮的关系
[下一步建议]    ← 围绕新目标的建议
```

## <a id="ix-06"></a>IX-06 状态定义与转换

### 全局系统状态

```
                    ┌──────────┐
          ┌────────→│  空闲态   │←──────────
          │         └────┬─────┘            │
          │              │ 用户输入仓库      │
          │              ▼                  │
          │         ┌──────────┐            │
          │         │  接入中   │            │
          │         └────┬─────┘            │
          │         ┌────┴────┐             │
          │       失败       成功            │
          │         │         │             │
          │         ▼         ▼             │
          │    ┌─────────┐ ┌──────────┐     │
          │    │ 接入错误 │ │  分析中   │     │
          │    └────┬────┘ └────┬─────┘     │
          │         │      ┌────┴────┐      │
          │    用户修改  分析失败  分析成功    │
          │    重新输入     │         │      │
          │         │      ▼         ▼      │
          │         │ ┌─────────┐ ┌──────┐  │
          └─────────┘ │分析错误 │ │对话态 │  │
                      └────┬────┘ └──┬───┘  │
                           │         │      │
                      用户修改   用户点击     │
                      重新输入   "切换仓库"   │
                           │         │      │
                           └─────────┴──────┘
```

### 状态详情

| 状态 | 描述 | 可用操作 | 界面视图 |
|------|------|---------|---------|
| 空闲态 | 无仓库，等待输入 | 输入仓库路径/URL | 视图 A |
| 接入中 | 正在验证仓库可访问性 | 无（等待） | 视图 B（前 1-2 步） |
| 接入错误 | 仓库路径/URL 无效或不可访问 | 修改输入，重试 | 视图 A + 错误提示 |
| 分析中 | 正在扫描和分析仓库 | 无（等待） | 视图 B |
| 分析错误 | 分析过程出错 | 重试或换仓库 | 视图 A + 错误提示 |
| 对话态 | 首轮报告已展示，多轮对话中 | 追问、切换目标、切换仓库 | 视图 C |

### 对话态子状态

```
对话态
  ├── 等待用户输入    ← 输入框可用
  ├── Agent思考中     ← 输入框禁用，显示思考指示器
  └── Agent回答展示中  ← 流式输出回答内容
```

### 组件状态

**输入框状态**：

| 状态 | 条件 | 表现 |
|------|------|------|
| 空闲 | 无内容 | 显示 placeholder |
| 有内容 | 用户已输入 | 发送按钮可用 |
| 禁用 | Agent 正在回答 | 置灰，显示"Agent 正在思考..." |

**下一步建议按钮状态**：

| 状态 | 条件 | 表现 |
|------|------|------|
| 可点击 | Agent 回答完毕 | 正常显示，有 hover 效果 |
| 禁用 | Agent 正在回答 | 不可点击 |

## <a id="ix-07"></a>IX-07 输入交互规格

### 仓库输入（视图 A）

| 交互要素 | 规格 |
|---------|------|
| 输入类型 | 单行文本输入 |
| placeholder | "输入本地仓库路径或 GitHub 公共仓库 URL" |
| 校验时机 | 失焦时 + 点击"开始"时 |
| 合法格式 | 绝对路径 或 `https://github.com/{owner}/{repo}` |
| 非法反馈 | 输入框下方红色文字提示具体原因 |
| 快捷操作 | Enter 键等同于点击"开始" |

### 对话输入（视图 C）

| 交互要素 | 规格 |
|---------|------|
| 输入类型 | 多行文本输入 |
| placeholder | "输入你的问题，或点击上方建议..." |
| 发送方式 | Enter 发送 / Shift+Enter 换行 |
| 发送按钮 | 输入非空时显示发送图标 |
| Agent 思考时 | 输入框禁用，placeholder 变为"Agent 正在思考..." |
| 空消息 | 不允许发送，发送按钮禁用 |

### 下一步建议点击

| 交互要素 | 规格 |
|---------|------|
| 展示形式 | 文字按钮/链接列表 |
| 点击行为 | 将建议文字填入输入框并自动发送 |
| 位置 | 每轮 Agent 回答末尾 |
| 数量 | 1-3 条 |

## <a id="ix-08"></a>IX-08 输出内容展示规格

### 首轮教学报告展示结构

首轮报告是一个特殊的长消息，内容分区展示：

```
┌─ 首轮教学报告 ─────────────────────────────┐
│                                            │
│  ## 仓库概览                                │
│  一句话说明这是什么项目                       │
│                                            │
│  ## 先抓什么                                │
│  当前仓库建议先关注的 2-4 个重点              │
```

## <a id="ix-09"></a>IX-09 异常与降级交互

### 错误场景交互

| 错误场景 | 触发条件 | 界面表现 | 用户可执行操作 |
|---------|---------|---------|-------------|
| 路径不存在 | 本地路径验证失败 | 视图 A 输入框下方红色提示："该路径不存在，请检查后重试" | 修改输入 |
| 路径不是目录 | 路径指向文件 | 红色提示："该路径不是目录，请输入仓库根目录路径" | 修改输入 |
| GitHub URL 不合法 | URL 格式错误 | 红色提示："GitHub URL 格式不正确，请使用 https://github.com/owner/repo 格式" | 修改输入 |
| 仓库不可公开访问 | GitHub API 返回 404/403 | 红色提示："该仓库不可访问，请确认仓库存在且为公开仓库" | 修改输入 |
| 分析超时 | 超过性能目标时间 | 分析视图中显示："分析时间较长，正在尝试降级分析..." | 等待 |
| 分析失败 | 内部错误 | 回到视图 A，显示："分析过程出错，请重试或尝试其他仓库" | 重试或换仓库 |

### 降级交互

| 降级场景 | 触发条件 | 交互表现 |
|---------|---------|---------|
| 大型仓库降级 | 源码文件 >3000 | 分析视图显示"仓库较大，优先输出结构总览和阅读起点"；首轮报告标注"部分内容因仓库规模限制未完成分析" |
| 非 Python 仓库降级 | 主语言非 Python | 首轮报告使用降级模板（见 IX-08），明确标注"当前语言暂不完整支持" |
| 入口未找到降级 | 无可靠入口候选 | 首轮报告入口候选区域显示"未找到可靠入口候选"，同时给出替代阅读建议 |
| 流程无法形成降级 | 证据不足 | 流程区域显示"当前证据不足以形成可靠流程骨架"，给出阅读建议替代 |

### 错误提示设计原则

1. **明确说明发生了什么**：不用模糊语言，直接告知用户具体问题
2. **明确说明用户可以做什么**：给出具体的修复建议
3. **不显示技术栈报错**：用户看到的是面向用户的语言，不是报错堆栈
4. **保持当前输入**：出错后不清空用户已输入的内容，方便修改

## <a id="ix-10"></a>IX-10 深浅控制交互

### 深浅级别定义

| 级别 | 触发指令 | 内容特征 |
|------|---------|---------|
| 浅 | "讲浅一点" | 少用术语；每个概念用类比解释；省略代码细节；每轮 1-2 个认知点 |
| 默认 | 首次进入 | 适度术语但有解释；展示关键代码路径名；每轮 2-4 个认知点 |
| 深 | "讲深一点" | 更多实现细节；展示代码片段；引用更多证据；但仍标注不确定项 |

### 深浅切换交互

```
用户："讲浅一点"
        │
        ▼
Agent 确认切换："好的，我用更简单的方式来讲。"
        │
        ▼
后续回答调整为浅层级
        │
        ├── 减少术语密度
        ├── 用类比替代术语
        ├── 减少代码引用
        └── 每轮认知点减少
```

### 深浅控制的持续性

- 深浅级别在切换后持续生效，直到用户再次切换
- 切换仓库后重置为默认级别
- "讲浅一点"和"讲深一点"可叠加（连续说两次"讲深一点"不额外加深，维持"深"级别）

### 深浅切换不影响的内容

- 不确定项标注：无论深浅都必须标注
- 证据引用：深度减少时可省略部分证据，但确定性结论仍需至少 1 条
- 下一步建议：始终保留
